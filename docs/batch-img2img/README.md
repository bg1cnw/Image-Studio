# 图生图批处理

本文档介绍桌面端图生图批处理模式的使用方式、行为边界和当前限制。

## 适用场景

适合这类需求:

- 你有一整个文件夹的素材图。
- 每张图都想套用同一组 prompt 和图生图参数。
- 你希望应用按并发逐张处理，而不是手动一张张点。

这不是现有“多参考图”或“出图张数”的另一种说法。

- 多参考图:一次 edit 请求里带多张参考图，服务于同一张结果。
- 出图张数:同一组参数重复发起多次请求，得到多张候选结果。
- 批处理:对目录里的每一张源图分别发起独立的图生图任务。

## 入口位置

在桌面端控制面板中:

1. 切换到 `图生图`。
2. 找到 `源图片 / 参考图` 区域。
3. 在顶部切换 `普通图生图 / 批处理`。

切到 `批处理` 后，手动参考图区会收起，改为批处理配置面板。

## 使用流程

1. 选择输入文件夹并扫描当前目录中的图片文件。
   或者直接选择多张图片，把它们加入当前批处理队列。
2. 确认当前批处理队列中的图片数量和示例文件名。
3. 输入 prompt，并按需要设置比例、质量、seed、negative prompt 等图生图参数。
4. 选择输出位置。
   - 默认保存回原目录:每张结果回到对应源图所在目录。
   - 独立输出路径:所有结果统一保存到你选定的目录。
5. 设置并发数。
6. 点击 `编辑` 开始处理。

## 输入规则

当前版本只处理你选择的这一层目录，不递归子目录。

如果你不想走目录模式，也可以直接选择多张图片加入批处理队列。

支持的输入图片格式:

- `png`
- `jpg`
- `jpeg`
- `webp`

非图片文件和子目录会被忽略。

## 输出规则

### 默认输出

如果不额外指定输出目录，每张结果会保存到各自源图所在目录。

例如:

- 源图: `/photos/a/cat.png`
- 结果默认保存到: `/photos/a/`

### 独立输出目录

如果你在批处理面板里选择了独立输出目录，则所有结果会统一保存到该目录。

### 命名规则

结果文件默认使用:

`processed-原文件名`

例如:

- `cat.png` -> `processed-cat.png`
- `portrait.webp` -> `processed-portrait.webp`

如果目标目录已经有同名文件，会自动避重:

- `processed-cat-2.png`
- `processed-cat-3.png`

## 并发与运行行为

- 每张源图都会创建一个独立的图生图任务。
- 并发数表示同时运行多少张，不是一次请求里上传多少张图。
- 实际可并发数量仍受当前上游 profile 的并发限制约束。
- 如果并发开得太高，应用可能会自动关闭流式预览，优先保证最终图完整。

## 不同比例源图的处理

如果一个目录里的源图宽高比不一致，但你仍然希望:

- 所有图片使用相同提示词
- 所有图片使用同一档分辨率
- 每张图仍然尽量保持各自源图的比例

可以在批处理面板里开启:

`按源图比例自动适配`

然后再选择统一的分辨率档位，例如 `1K`、`2K`、`4K`。

这时应用会对每一张源图分别计算:

- 比例:取该源图自身宽高比
- 分辨率:取你在批处理面板里指定的统一档位，并沿用应用当前对 `1K / 2K / 4K` 的尺寸换算规则

例如:

- 横图会得到横向尺寸
- 竖图会得到纵向尺寸
- 它们都仍然共享同一个提示词和同一个分辨率档位

这里的 `1K / 2K / 4K` 仍然遵循应用当前的尺寸语义:

- 内置比例会落到内置尺寸矩阵
- 非内置比例会按该档位的像素面积参考值推导具体宽高

注意:

- 这是批处理模式下的额外尺寸策略。
- 它不会改变普通图生图里 `Auto` 比例与 `size=auto` 的原有语义。

## 参数继承

批处理会复用你当前图生图控制面板里的参数，包括:

- prompt
- 比例 / 尺寸
- 质量
- 输出格式
- seed
- negative prompt
- 背景、压缩率、审核等高级参数

对于混合比例素材，推荐保持 `Auto` 或按你的上游能力选择更宽松的尺寸策略。

## 当前限制

- 仅支持桌面端。
- 仅扫描当前目录，不递归子目录。
- 目录中的每张图都使用同一套参数，不支持按文件单独覆盖。
- 批处理模式的输出文件会额外保存到目标目录，但应用内部历史仍会保留正常的生成记录，便于详情查看、历史筛选和再次复用。

## 什么时候不要用它

下面这些场景更适合继续用普通图生图:

- 你只想编辑 1 到 2 张图。
- 你需要手动挑多张参考图共同作用到一张结果上。
- 你希望每张图都用不同的 prompt 或不同的参数。
