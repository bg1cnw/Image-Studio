# 隐私政策

最后更新：2026-06-06

本隐私政策适用于本仓库当前提供的 Image Studio 客户端，包括桌面版（Windows、macOS、Linux）与 Android 壳层版本。若你使用仓库中的浏览器预览/调试环境，部分本地存储行为会与正式客户端不同，本文也会一并说明。

## 1. 基本原则

- Image Studio 不提供开发者自营的内置生图服务。
- 你输入的提示词、参考图、蒙版、模型参数等数据，默认只会保存在你的设备本地，或发送到你自己配置的上游 `BASE_URL`。
- 当前仓库代码中未集成广告 SDK、统计埋点 SDK、崩溃上报 SDK，也不会自动把你的提示词、图片或 API Key 上传到开发者控制的独立采集服务器。

## 2. 我们会处理哪些数据

Image Studio 在本地可能处理以下数据：

- 上游配置与偏好设置：例如 `BASE_URL`、API 形态、文本/图像模型 ID、请求策略、并发限制、代理模式与代理地址、主题、字号、输出格式、预设、自定义比例、输出目录、受信任输出目录、是否保留日志、忽略的版本号、完成提示音配置等。
- API Key：
  - 桌面版默认使用系统安全存储，不写入普通 JSON 配置。
  - Android 版保存在应用私有 `SharedPreferences` 中。
  - 浏览器预览环境没有系统密钥库时，会退回浏览器本地存储，仅适合调试。
- 生成与历史数据：例如提示词、优化后提示词、模式、尺寸、质量、风格、种子、负面提示词、源图片路径、生成结果路径、原始响应日志路径、历史时间戳等。
- 本地图片副本：为了支持拖拽导入、剪贴板粘贴、历史预览、旋转/翻转/裁剪、时间线回看等功能，应用可能在本地保存导入图片、副本、缩略图、预览图或历史全图数据。
- 可选用户标识：如果你主动填写“用户标识”，该值会随请求发往你配置的上游，用于上游侧的安全审计或滥用防护。Responses API 下映射为 `safety_identifier`，Images API 下映射为 `user`。

## 3. 本地存储位置

### 桌面版

- Windows：
  - 默认输出目录通常为 `Documents\\Image Studio`
  - 生成图片位于 `images/`
  - 原始响应日志位于 `log/`
  - 缩略图/预览图位于 `thumbs/`、`previews/`
  - 兼容状态文件位于 `compat/state.json`
  - 导入图片与本地编辑派生文件位于 `imports/`
  - WebView2 本地数据位于 `webview/`
- macOS：
  - 默认输出目录通常为 `~/Pictures/Image Studio`
  - 应用状态、导入文件等通常位于 `~/Library/Application Support/image-studio/`
- Linux：
  - 默认输出目录通常为 `~/Pictures/Image Studio`
  - 应用状态、导入文件等通常位于 `~/.config/image-studio/`

### Android 版

- 默认输出目录通常为应用私有目录 `Android/data/<应用包名>/files/Pictures/ImageStudio`
- 导入图片副本通常位于应用私有目录下的 `files/imports`
- API Key 与输出目录等配置保存在应用私有 `SharedPreferences`
- 在 Android 10 及以上版本，如果你执行“保存图片”操作，图片可能写入系统相册 `Pictures/ImageStudio`

### 浏览器预览/调试环境

- 历史与图片缓存可能保存在浏览器 `IndexedDB`（数据库名为 `image-studio`）
- 设置项可能保存在浏览器 `localStorage`（键名前缀通常为 `gptcodex.`）
- 在无原生宿主能力时，浏览器预览环境可能用 `localStorage` 暂存 API Key；这不是正式客户端的安全存储方案

## 4. 何时会发生网络请求

### 4.1 发送到你配置的上游

当你执行以下操作时，应用会把相关数据发送到你配置的 `BASE_URL`：

- 测试连接
- 文生图
- 图生图
- Prompt 优化

可能发送的数据包括：

- API Key（作为 `Authorization: Bearer ...` 请求头）
- 提示词、参考图、蒙版
- 模型 ID、尺寸、质量、输出格式、风格、种子、负面提示词等生成参数
- 你主动填写的用户标识

这些请求是直接发往你配置的上游，或按你的代理设置经代理转发。上游服务与代理服务如何处理这些数据，不受 Image Studio 控制，你应自行评估其隐私与合规政策。

### 4.2 桌面版更新检查

桌面版当前版本支持检查新版本。检查更新时，会向 GitHub 请求：

- `https://api.github.com/repos/RoseKhlifa/Image-Studio/releases/latest`

这会使 GitHub 获取你的网络请求元数据，例如 IP 地址、请求时间、User-Agent 或其他标准 HTTP 信息。

### 4.3 外部链接

当你主动打开发布页、Issues 或其他外部链接时，系统浏览器会访问对应站点；相关站点会按其自身政策处理访问数据。

## 5. 不会自动上传什么

在当前仓库实现中，应用不会自动执行以下行为：

- 不会把你的 API Key 上传给开发者自建统计平台
- 不会把你的提示词或生成图片自动同步到开发者服务器
- 不会集成第三方广告网络
- 不会内置崩溃上报或行为埋点 SDK

但请注意：

- 你主动配置的上游服务本身可能记录请求日志
- 你主动导出历史、保存图片、复制到外部目录后，相关文件将离开应用私有范围

## 6. 日志、原始响应与排错数据

桌面版会在生成期间把上游原始响应写入输出目录下的 `log/` 子目录，以便你排查问题，例如：

- Responses API 的 SSE 文本响应
- Images API 的 JSON 响应

关于日志保留：

- 如果你开启“保留日志”，这些文件会继续保留在本地
- 如果你关闭“保留日志”，桌面端会在应用关闭时清理受管理的日志目录
- 即使关闭“保留日志”，在应用运行期间，当前请求的临时原始响应文件仍可能短暂存在，直到应用退出并执行清理

## 7. Android 权限与系统备份

Android 版当前声明的主要权限包括：

- `INTERNET`：用于访问你配置的上游服务
- `READ_EXTERNAL_STORAGE`（仅 Android 12 及以下生效）：用于读取你选择的本地图片

另外，Android 清单当前启用了系统备份能力：

- `android:allowBackup="true"`
- 当前备份规则排除了数据库，但没有排除全部应用私有文件与 `SharedPreferences`

这意味着在你的 Android 系统允许云备份或设备迁移时，以下数据可能随系统机制参与备份/迁移：

- 本地配置
- Android 端保存的 API Key
- 应用私有目录中的导入文件或其他本地文件

是否实际发生备份，还取决于你的 Android 系统版本、账户设置、加密能力与设备迁移流程。

## 8. 数据安全

- 桌面版 API Key 默认写入系统密钥库（如 Keychain、Credential Manager、Secret Service）
- 桌面版兼容状态文件与多数受管理文件使用仅当前用户可读写的权限创建
- Android 私有目录与 `SharedPreferences` 受 Android 应用沙箱保护

但你仍需理解以下风险：

- 你选择的上游或代理可能位于第三方服务器
- 你主动保存到公共目录、系统相册或导出到其他位置的文件，可能被其他应用、系统服务或具有相应权限的用户访问
- 浏览器预览环境的安全性低于正式桌面客户端

## 9. 你的控制权

你可以通过以下方式控制本地数据：

- 在应用设置中修改或清空 API Key
- 在应用中清空历史记录
- 关闭“保留日志”
- 修改输出目录
- 手动删除输出目录中的图片、日志、缩略图与导出的历史文件
- 手动删除应用状态目录中的 `compat/state.json`、`imports/`、浏览器存储或 WebView 数据
- 在 Android 中使用“清除存储”或卸载应用删除应用私有数据
- 单独删除你已经保存到系统相册、公共目录或外部位置的图片与 JSON 文件

## 10. 历史导入与导出

当你主动使用历史导出功能时：

- 桌面版会通过系统保存对话框写出 JSON 文件
- Android 版会把导出的 JSON 写到当前输出目录

当你主动导入历史文件时，文件内容会被读入本地并合并到本地历史中。导入/导出均属于用户主动操作，不会自动执行。

## 11. 适用范围变更

如果后续版本引入新的联网能力、账户系统、同步服务、统计服务或新的第三方 SDK，本文件应随代码行为一起更新。
